# Mobile Asset Intake

## Folder Layout

```
assets/
├── brand/                  # Logo, app icon source, splash source
│   └── logo.svg            # Master logo (drop here first)
├── figma/
│   ├── screens/            # Full-screen PNG/SVG exports from Figma
│   ├── components/         # Exported reusable component snapshots
│   └── tokens.json         # Design tokens (Figma Tokens plugin export)
└── _manifest.json          # Auto-generated by `pnpm assets:ingest`
```

## Naming Convention

- **Screens:** `<flow>__<screen>.png` — e.g. `onboarding__archetype_picker.png`
- **Components:** `<component_name>.png` — e.g. `button_primary.png`
- **Brand:** descriptive, no spaces — e.g. `logo.svg`, `icon_foreground.svg`

## Workflow

1. Drop Figma exports into the appropriate subfolder.
2. Run `pnpm assets:ingest` — optimises images, produces `@2x`/`@3x` variants, updates `_manifest.json`.
3. Run `pnpm tokens:sync` — regenerates Tamagui theme from `tokens.json`.
4. Reference `_manifest.json` paths in UI-generation tasks.

## Design Token Schema

`tokens.json` follows the [Figma Tokens plugin](https://docs.tokens.studio/) format:

```json
{
  "global": {
    "color": { "brand": { "blue": { "500": { "value": "#1E6FD9", "type": "color" } } } },
    "spacing": { "4": { "value": "16", "type": "spacing" } }
  }
}
```

## Commit Rules

- Run `pnpm assets:ingest` before committing new assets — the Husky pre-commit hook will reject un-ingested files.
- PNG/SVG files are tracked as Git LFS objects (see root `.gitattributes`).
